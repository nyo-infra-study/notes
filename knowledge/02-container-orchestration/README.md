# Container Orchestration

[← Back to Knowledge Base](../README.md)

## Overview

Learn how applications run in clusters and how to manage them at scale. Kubernetes is the platform where everything runs, and Helm is how you package and deploy applications onto it. These two tools form the foundation of the entire deployment pipeline.

## Learning Path

1. **[Kubernetes](./01-kubernetes.md)** - Pods, Services, Ingress, and Deployments — the platform where everything runs
2. **[Helm](./02-helm.md)** - Charts, Templates, and Values — package and manage your applications

## Prerequisites

Before diving into this section, you should understand:
- [Foundations](../01-foundations/README.md) — text manipulation for reading YAML, system monitoring for debugging pods

## What You'll Learn

- How containerized applications are scheduled and run in a cluster
- How to expose services internally and externally
- How to template Kubernetes manifests and manage configuration across environments

## Estimated Time

⏱️ 4-6 hours total
