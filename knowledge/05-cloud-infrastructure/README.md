# Cloud Infrastructure

[← Back to Knowledge Base](../README.md)

## Overview

AWS services that power and support your Kubernetes infrastructure. Your cluster doesn't run in a vacuum — it sits on top of compute, storage, networking, and security primitives provided by AWS. This section is a reference guide to those services and how they relate to what you've already learned.

## Learning Path

1. **[Compute](./01-compute.md)** - EC2, Lambda, and ECS — where workloads run
2. **[Storage](./02-storage.md)** - S3, EBS, and EFS — persistent data solutions
3. **[Database](./03-database.md)** - RDS, DynamoDB, and ElastiCache — managed database services
4. **[Networking](./04-networking.md)** - VPC, Route53, and CloudFront — connect and expose services
5. **[Security and IAM](./05-security-and-iam.md)** - IAM, KMS, and Secrets Manager — access control and encryption
6. **[Management and Governance](./06-management-governance.md)** - CloudWatch, CloudTrail, and Config — monitoring and compliance

- [Foundations](../01-foundations/README.md)

## What You'll Learn

- How the AWS services your cluster depends on are structured and configured
- How IAM roles and policies control access to cloud resources
- How networking primitives like VPCs and security groups protect your infrastructure
- How to use CloudWatch and CloudTrail for monitoring and audit trails
